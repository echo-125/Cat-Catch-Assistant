# 猫抓助手 (Cat Catch Assistant)

一个功能完善的 M3U8 视频下载工具和自动嗅探助手，支持多任务并行下载、断点续传、自动重试等功能。

## ✨ 功能特点

### 核心功能
- ✅ 自动解析 M3U8 播放列表（支持 .jpg 等伪装扩展名）
- ✅ 多线程并发下载，速度快
- ✅ 支持断点续传
- ✅ 失败自动重试（最多3次）
- ✅ 实时下载速度和剩余时间显示
- ✅ 自动合并 TS 分片
- ✅ 自动转换为 MP4 格式（需要 ffmpeg）

### 多任务管理
- ✅ 支持多任务并行下载
- ✅ 任务队列管理
- ✅ 实时进度显示
- ✅ 任务状态跟踪（等待中、下载中、已完成、已失败、已暂停）

### 用户体验
- ✅ 图形界面支持，操作简单
- ✅ 配置自动保存和加载
- ✅ 繁体中文自动转简体
- ✅ 完整的中文注释，适合学习

## 📦 安装依赖

### 1. 安装 Python 依赖

```bash
pip install -r requirements.txt
```

### 2. 安装 ffmpeg（可选，用于转换 MP4）

**Windows:**
```bash
# 使用 Chocolatey
choco install ffmpeg

# 使用 Scoop
scoop install ffmpeg

# 或手动下载: https://www.gyan.dev/ffmpeg/builds/
```

**Mac:**
```bash
brew install ffmpeg
```

**Linux:**
```bash
sudo apt install ffmpeg  # Ubuntu/Debian
sudo yum install ffmpeg  # CentOS/RHEL
```

## 🚀 使用方法

### 方式一：图形界面（推荐）

```bash
python m3u8_downloader_gui.py
```

**操作步骤：**
1. 输入 M3U8 链接
2. 选择下载路径（自动记住）
3. 输入文件名（可选，支持繁体自动转简体）
4. 设置并发线程数（自动记住）
5. 点击"添加到下载列表"
6. 选择任务，点击"开始选中"或"全部开始"
7. 查看实时进度和日志

### 方式二：命令行

```bash
python m3u8_downloader.py <M3U8_URL> [输出文件名] [保存路径]
```

示例：
```bash
python m3u8_downloader.py http://example.com/index.jpg my_video ./downloads
```

### 多任务并发流程

```
用户添加多个任务
        ↓
┌───────────────────┐
│  任务队列管理器   │
│  max_concurrent=3 │
└───────────────────┘
        ↓
    ┌───────┐
    │ 任务1 │ ← 下载中（线程1）
    ├───────┤
    │ 任务2 │ ← 下载中（线程2）
    ├───────┤
    │ 任务3 │ ← 下载中（线程3）
    ├───────┤
    │ 任务4 │ ← 等待中
    ├───────┤
    │ 任务5 │ ← 等待中
    └───────┘
        ↓
  任务1完成 → 任务4开始
  任务2完成 → 任务5开始
```

### 断点续传流程

```
检查分片文件是否存在
        ↓
    存在？ ──Yes──→ 跳过下载，计入已完成
        │
        No
        ↓
    下载分片
        ↓
    成功？ ──Yes──→ 保存文件，更新进度
        │
        No
        ↓
    重试次数 < 3？
        │
        Yes ──→ 等待2秒，重新下载
        │
        No
        ↓
    标记为失败
```

## 📊 性能优化

### 1. 并发下载
- 使用 `ThreadPoolExecutor` 线程池
- 默认16线程，可调整1-64
- 显著提升下载速度

### 2. 断点续传
- 检查文件是否已存在
- 跳过已下载的分片
- 支持中断后继续

### 3. 失败重试
- 每个分片最多重试3次
- 重试间隔2秒
- 提高下载成功率

### 4. 实时统计
- 下载速度（片/秒）
- 剩余时间估算
- 进度百分比

## ❓ 常见问题

### 1. 如何获取M3U8链接？

**推荐方法：**
使用Chrome浏览器插件（如 cat-catch、Video DownloadHelper等）：
1. 安装插件
2. 打开视频网页
3. 播放视频
4. 使用插件的"深度搜索"功能
5. 复制找到的M3U8链接

**手动方法：**
1. 打开Chrome开发者工具（F12）
2. 切换到 Network 标签
3. 过滤：`m3u8` 或 `index`
4. 播放视频
5. 查找 `.m3u8` 或 `index.jpg` 请求
6. 右键 → Copy → Copy link address

### 2. 提示 "未找到 ffmpeg"

**解决方法:**
- 安装 ffmpeg 并添加到系统 PATH
- 或手动转换: `ffmpeg -i merged.ts -c copy output.mp4`

### 3. 下载速度慢

**解决方法:**
- 修改并发线程数（默认16，可增加到32或64）
- 在GUI中调整"并发线程"参数

### 4. 部分分片下载失败

**解决方法:**
- 程序会自动重试3次
- 检查网络连接
- 检查 M3U8 链接是否有效

### 5. 繁体中文显示问题

**解决方法:**
- 程序自动将繁体转换为简体
- 转换过程会在日志中显示

## 📝 技术要点

### M3U8 文件格式

```m3u8
#EXTM3U                    # 文件头
#EXT-X-VERSION:3           # 版本号
#EXT-X-TARGETDURATION:10   # 最大分片时长
#EXTINF:9.9,               # 分片时长
segment0.ts                # 分片文件名
#EXTINF:9.9,
segment1.ts
#EXT-X-ENDLIST             # 结束标记
```

### URL 处理

- 相对路径自动转换为完整 URL
- 使用 `urllib.parse.urljoin()` 处理

### 编码支持

- UTF-8（默认）
- Big5（繁体中文）
- GBK/GB2312（简体中文）
- 自动检测和转换

### 线程安全

- 使用消息队列（Queue）通信
- 避免界面卡顿
- 正确处理并发
